# 回测性能优化方案 (未实施)

## 1. 缓存指标计算

**问题**：每步对每个 symbol/interval 执行 `df[df.index <= ts].copy()` + `compute_all()`，50K 步 × 25 组合 = ~125 万次切片和指标重算。

**方案**：回测启动时一次性完成所有指标计算，存入带索引的 DataFrame。每步只需 `df.loc[ts]` 查找。

**修改位置**：
- `core/backtest/engine.py` — `run_with_exit_evaluation()` 启动阶段
- 新增 `_precompute_indicators()` 方法

**收益**：消除 ~125 万次切片+compute_all，预计节省 40-50% 每步耗时。

**代价**：内存从 ~50MB 增至 ~200MB（存储额外指标列）。

---

## 2. 加长 ML 重训练间隔

**问题**：当前每 50 根 K 线重训练 XGBoost，全回测 ~1,000 次训练。

**方案**：`ml_retrain_interval` 从 50 改为 100。

**修改位置**：
- `core/backtest/engine.py:126` — `ml_retrain_interval = 50` → `= 100`

**收益**：训练次数减半（~1,000 → ~500），每次多 50 行样本，模型更稳定。

**精度影响**：可忽略。金融统计特性不是每 50 分钟就变的。100 根（1h40m）仍足够捕捉日内趋势。

---

## 3. 合并时间轴优化 (注意风险)

**问题**：15m/1h 策略在 1m 时间轴 95% 的时间步上，K 线无变化，入场检查浪费。

**方案**：保持统一时间轴用于 SL/TP 和 equity curve。入场检查时，15m/1h 策略只在有新 K 线的时间步执行。

**风险**：需确保跨策略一致性不受影响。改动较大，需要充分测试。

**优先级**：低。前两项优化已足够显著。

---

## 4. 多策略并行训练

**问题**：30 个策略×交易对组合的 ML 训练当前是串行的。

**方案**：使用 `concurrent.futures.ThreadPoolExecutor` 并行训练 30 个模型。

**收益**：在空闲核心上并行，但当前 CPU 已 99%，提升有限。

**优先级**：低。等前两项优化后 CPU 有空余再考虑。
